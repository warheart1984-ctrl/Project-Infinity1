# Domain Binding — Mythar / SRE under Civilization OS

**Status:** Boundary charter  
**Date:** 2026-07-20  
**Law:** Drive-G-1

---

## Hierarchy (frozen)

```
Civilization OS (Project Infi / AAES substrate)
└── Domain: Linguistic Reconstruction
    ├── Framework (open posture): CIH · LRL · CRA · FRA · CEL
    ├── Runtime (reference): SRE — G:\sovereign-reconstruction-engine
    └── Product IP (proprietary): Mythar — lexicon, grammar, cosmology, commercial licenses
```

**Infi/AAES is the civilization substrate.**  
**Mythar is a governed language world (domain product).**  
**SRE is a conformant reconstruction + evidence runtime for that domain — not a rival OS.**

---

## What each layer owns

| Layer | Owns | Does not own |
|-------|------|--------------|
| **Infi / AAES** | Organism identity, law spine, governed memory, operator surfaces, civilization receipts | Mythar lexicon authorship; Mythar commercial license |
| **CIH / LRL / CRA / CEL / FRA** | Constitutional reconstruction *framework* (open specs) | Mythar proprietary content |
| **SRE** | Reference implementation of FRA/CIH/CEL for reconstruction workloads | Civilization OS kernel; Mythar IP |
| **Mythar** | Language artifacts + licensing | Civilization OS; CIH specification |

## How they should connect (planned)

| Integration | Direction | Notes |
|-------------|-----------|-------|
| Domain intent | Infi operator / platform → SRE API | e.g. start reconstruction, fetch lineage |
| Evidence | SRE CEL / certificates → Infi receipt / evidence packages | Map CEL entry → civilization receipt id |
| Law | Infi law gate may require “domain certificate present” | Do not re-implement CIH inside Infi |
| Lexicon product | Mythar licenses remain Mythar commercial docs | Infi must not open-source Mythar by accident |

Vertical slice v0.1 (**LIRL**) does **not** require Mythar. Domain plug-in is **slice v0.2+**.
