# Release Criteria — Steward Council (v1.0)

**Authority:** RFC-COR-SUITE-1.0 · [COR-Suite-Spec-1.0.md](../spec/COR-Suite-Spec-1.0.md)

---

## 1. Inputs Required

A release decision **MUST** consider:

- CAV-1.0 validation report
- COR-1.0 state vector
- CSR-1.0 stewardship report
- DRA-1.0 dependency-risk assessment
- Proof Analysis results
- constitutional invariants
- CAR-1.0 lifecycle status

---

## 2. Blocking Conditions

Release **MUST NOT** proceed if any are present:

- invalid canonical artifacts
- broken provenance chains
- missing required canonical artifacts
- schema violations
- unresolved structural integrity failures
- unverified critical requirements
- governance receipts missing signatures

---

## 3. Advisory Conditions

Release **MAY** proceed but **MUST** be documented:

- research claims
- high dependency-risk nodes
- deprecated authorities with successors
- non-critical verification gaps
- incomplete maturity for non-critical requirements

---

## 4. Required Steward Actions

Stewards **MUST**:

- document rationale
- reference evidence
- enforce invariants
- sign governance receipt
- register decision in CAR-1.0

---

## 5. Decision Types

- Approve
- Reject
- Require Fixes
- Escalate
- Freeze
- Retire
